## MCP (Model Context Protocol)

Ранее был «стадией 8» дорожной карты — **убран из плана MVP**. Суть — два разных MCP,
и оба надстройки, а не ядро:

1. **MCP для анализа уже внесённых данных.** Отдаёт ИИ-ассистенту программный доступ к
   тому, что приложение **уже** положило в CRM — работает **со смарт-процессом**
   (элементы платежей/разнесения, статусы импорта). ИИ отвечает на вопросы («какие
   приходы за период», «что не разнеслось», «статус последнего импорта») и участвует в
   триаже обратной связи. Живёт поверх нашего домена — не трогает банк.

2. **MCP для прямой работы с клиент-банкингом.** Программный доступ к самому банковскому
   API (запрос выписки/операций напрямую). Это **отдельное приложение**, не часть этого
   репозитория: у него свой контур авторизации/безопасности (банк-креды), свой скоуп
   ответственности; смешивать с импорт-приложением не следует.

**Почему вынесено:** для запуска не нужно; тип 1 — приятная надстройка после MVP, тип 2 —
вообще отдельный продукт. Вернуть в план — только по явному запросу владельца.

## Прочее (по мере спроса)

- Ещё стандартные форматы выписки: camt.053, MT940, CSV (формат 1С уже поддержан).
- Сопоставление платежей с заказами/инвойсами 1:1 с сохранением в смарт-процесс —
  сверх текущего разнесения (если тариф портала поддерживает смарт-процессы).
