# Карта продукта — демо на лендинге

> Last reviewed: 2026-07-17

Рабочий журнал и карта задачи «демо-версия на публичном лендинге». Сверяюсь с ним перед
каждым куском работы и держу синхронно с [`project-map.md`](project-map.md) и [`CLAUDE.md`](../CLAUDE.md).

## Цель (что просил владелец)

1. **Демо разбора файла на лендинге.** Пользователь прикрепляет файл выписки → получает
   ответ: *что в файле* и *какие данные извлечём* (операции, контрагент, суммы, назначение,
   распознанные идентификаторы). Разбор — **в браузере**, детерминированный (без backend/AI).
2. **Рабочий пример «песочницы» к Альфе и Приору** на демо-странице. Кнопки прогоняют
   **реальные нормализаторы** (`normalizeAlfa`/`normalizePrior`) на образцах ответов
   sandbox-формата → тот же экран извлечения. Это демонстрирует весь путь «ответ банка →
   нормализованные операции» без обращения к банку (демо на лендинге не держит банк-креды —
   живой OAuth идёт из портала, backend).
3. Дизайн/тексты/уведомления — «потом поправим». Приоритет: **всё должно отработать**.

## Принятые решения

- Демо — **секция на существующем лендинге** (`app/pages/index.vue`, layout `landing`,
  тёмная брендовая тема), а не отдельная страница — одна точка входа, меньше дубля.
- Вся логика извлечения — в чистом `app/utils/demoExtract.ts` (покрыт тестами):
  образцы Альфа/Приор, сводка извлечения, распознавание идентификаторов по матрицам.
- Файловый разбор переиспользует готовое ядро `importUpload.ts` (`processUploadBatch`,
  windows-1251, дедуп) — то же, что на `/import`.
- Распознавание — реальный `recognizeByMatrices` (`purposeMatch.ts`) на демо-наборе матриц.
- Рендер результатов — самодостаточный тёмный UI внутри `LandingDemo.vue` (не тащим
  b24ui-`OperationList`, чтобы не смешивать light/dark-токены с брендовой оболочкой).

## Доработка по запросу владельца (кастом-посыл, предупреждение, доступы)

- **Обязательный посыл про кастом** — под демо голубой блок `LANDING_DEMO.customDev` +
  кнопка «Обсудить доработку» (→ `#brief`, цель `demo_custom_dev`): «дорабатываем под ваши
  процессы и разворачиваем на вашем сервере: ваш git, ваш контур».
- **Предупреждение о данных** — в демо `⚠️`-блок `LANDING_DEMO.privacyWarning`: «подумайте, что
  загружаете… не раскрывайте лишнюю чувствительную информацию». Заодно смягчает webvisor-риск.
- **Логин/пароль песочницы Альфа/Приор — НЕ публикуем** (осознанное решение): (1) значений у меня нет
  (git-ignored `.env.alfabankby`/`.env.priorbank`); (2) демо не логинится в банк — гоняет готовые
  образцы через реальные нормализаторы, логин не нужен; (3) публиковать банковские sandbox-доступы на
  публичном лендинге рискованно (абьюз песочницы) и противоречит только что добавленному предупреждению
  «не раскрывать лишнее». Если владелец захочет — пусть пришлёт значения и подтвердит, куда (публично /
  в закрытую доку). Уточняющий вопрос задать не удалось (сбой тула), выбран безопасный дефолт.
