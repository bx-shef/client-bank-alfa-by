# Калькулятор цены внедрения и сопровождения

> Last reviewed: 2026-07-18

Модель расчёта стоимости, сроков и сопровождения кастомных внедрений. Рабочий
процесс: **ты присылаешь требования → я считаю по этой модели** (деньги, время,
пакет сопровождения). Документ — источник правды по ставкам и коэффициентам;
меняем цифры здесь, расчёты подстраиваются.

**Ставки (не единая — по типу работы), RUB:**

| Работа | ₽/час |
|---|---:|
| Разработка (кастом-код) — **якорь** | 3 000 |
| Аналитика / ТЗ | 2 100 |
| Деплой / DevOps (по готовому шаблону) | 2 600 |
| Сопровождение (ретейнер) | 3 000 (срочное ×1.5) |

- **Модель цены внедрения:** гибрид — оцениваю часы по требованиям, умножаю на
  ставку по типу работы, отдаю **фикс-цену с вилкой** ±15%.
- **Сопровождение:** почасовой ретейнер **пакетами часов/мес**.
- **Все цены — «к оплате»** (с учётом налогов/эквайринга), не «на руки».
- **Минимальный чек проекта:** `30 000 ₽` (ниже не берёмся — не окупает
  переключение/коммуникацию).
- **Продуктивных часов в день:** `5 ч`.

## 1. Как считается цена (формула)

1. **Оценка часов по задачам.** Каждую задачу оцениваю по трём точкам
   (оптимистично `O`, реально `M`, пессимистично `P`) и свожу по PERT:
   ```
   Часы(задача) = (O + 4·M + P) / 6
   ```
   Это снимает «оптимизм» одной цифры — заложен риск разброса.
2. **База.** `БазаЧасов = Σ Часы(задача)` по всем задачам заявки (включая анализ/ТЗ).
3. **Накладные** (на разработческую/деплойную часть, кроме анализа):
   - тестирование и багфикс — **+20%**;
   - менеджмент/коммуникация — **+10%**;
   - риск-буфер (contingency) — **+15%** (при мутных требованиях — до +25%).
   ```
   ЧасыСНакладными = БазаЧасов × (1 + 0.20 + 0.10 + 0.15) = БазаЧасов × 1.45
   ```
   **Мелкая стандартная задача** (напр. типовой деплой): облегчённые накладные
   **×1.35** (тест +15%, PM +5%, риск +15%) — крупный PM/тест-оверхед не нужен.
4. **Коэффициенты** (перемножаются, применяются к итогу):
   | Условие | ×коэффициент |
   |---|---|
   | Развёртывание в контуре клиента (коробка / on-prem / чужой сервер) | ×1.15 |
   | Срочность (сжатые сроки, сверхурочно) | ×1.3–1.5 |
   | Настоящий кастом (нестандартные процессы/правила с нуля) | ×1.2 |

   > **Не двойным счётом.** «Коробка/on-prem» и «чужой сервер» — это **одно**
   > условие (деплой в контур клиента), берём коэффициент **один раз**, не
   > перемножаем.
   >
   > **Наценка ×1.2 — только на кастом.** Интеграция с банком (или другим
   > **стандартизованным** интерфейсом) — промышленный стандарт: пусть сложно, но
   > предсказуемо, наценки за неопределённость **не ставим**. Сложность банка
   > (напр. DCR/ключи Приора) идёт в **менеджмент + растянутый календарь**, а не в
   > часы прогера и не в риск-множитель. ×1.2 — для «кастома от Вани», где нет
   > промышленного стандарта.
5. **Стоимость.** Каждую категорию часов умножаю на **свою ставку** (аналитика
   2100 / разработка 3000 / деплой 2600), суммирую, округляю вверх. Не ниже
   минимального чека `30 000 ₽`.
6. **Подача клиенту.** Фикс-цена + **вилка ±15%**. Вилку сужаем после согласования
   ТЗ. Типовые работы отдаём **фикс-пакетом** (см. §2а) — без почасовой сметы.

### 2а. Готовые фикс-пакеты (продуктизировано)

Повторяемые работы — фикс-цена, без сметы (у нас уже есть `docker-compose.prod.yml`,
health-check, CSP-сборка — деплой не «с нуля»):

| Пакет | Что входит | Цена (к оплате) |
|---|---|---:|
| **Стандартный перенос на сервер** (первичный) | Приложение в контуре клиента: env, домен, TLS, токены, OAuth банка, вебхуки B24, смоук-тест | **~35 000 ₽ (≈ 1 310 Br)** |
| **Редеплой обновления** | Клиент уже на своём сервере — выкатываем обновлённый образ (новый формат/банк/правка) + смоук | **~8 000 ₽ (≈ 290 Br)** |
| **Разбор нового формата файла** | Парсер+нормализатор+тесты. **Продуктовое улучшение** — раскатывается на все серверы (~за месяц); принцип «поддержим любой распространённый формат» | **0 ₽ (бесплатно)** |
| **Локальный разворот кастом-формата** | Только если формат нужен клиенту **раньше** общей раскатки / на изолированном сервере — платим за разворот, не за формат | редеплой ~8 000 ₽ / первичный ~35 000 ₽ |
| **Источник файла** (за тип) | Откуда берём файл: email (IMAP/POP) / Telegram / чат B24 / FTP + расписание опроса. Берём **с каждого клиента** (это ценность ему); что написали раз — **наша маржа** | **~800 Br (~21 000 ₽)** |
| **Источник файла с ключами** (SFTP / SSH) | Как выше + управление ключами доступа | **~1 000 Br (~26 700 ₽)** |
| **Новый банк — понятный OAuth** | Разработка all-in (адаптер OAuth по нашим ядрам + нормализатор + fetch + тесты + реестр). Разворот — отдельно | **разработка ~42 700 ₽ (≈ 1 600 Br)** |
| **Новый банк — сложный (DCR/ключи, «Приор-класс»)** | Разработка ~2 000 Br + **менеджмент/ключи** (DCR-регистрация, получение/разворот ключей, consent, sandbox) ~1 800 Br + риск +10%. Разворот — отдельно | **~147 000 ₽ (≈ 5 500 Br)** с первичным разворотом |

> **Первый** перенос на новом типе окружения может быть дороже (строим шаблон);
> **повторные** — по фикс-пакету. «Сырой»/нестандартный сервер клиента → +часы
> по смете сверху.
>
> **Новый банк:** цена скачет от **авторизации**. Понятный OAuth (токен по кредам)
> — разработка ~1 600 Br. Сложный (DCR + подпись + ключи, как Приор) — разработка
> растёт немного (~2 000 Br, стандарт же), но добавляется **менеджмент + ключи**
> (~1 800 Br) и **растянутый календарь** (ждём банк). Банк ≠ кастом — наценки ×1.2
> нет.
>
> **Формат ≠ источник.** Сам **разбор формата — бесплатно** (продукт, раскатка на
> всех). Платное вокруг файла — это **источник** (откуда и как часто берём файл:
> email/TG/чат/FTP/SFTP/SSH + расписание) и **локальный разворот**, если нужно
> раньше общей раскатки. Источник берём **с каждого клиента по прайсу (~800 Br)** —
> это ценность именно ему. То, что транспорт написан **один раз**, — **наша маржа**,
> а не скидка клиенту (даром не отдаём).

## 3. Сроки

- **Продуктивных часов в день:** закладываю `5 ч` (остальное — контекст,
  коммуникация, паузы).
- `Рабочие дни = ЧасыИтог / 5`, сверху **буфер +20%** на согласования/приёмку.
- Календарные сроки зависят от параллельности и ответов клиента — даю диапазон,
  не дату.

## 4. Сопровождение (ретейнер пакетами часов/мес)

| Пакет | Часов/мес | Цена/мес |
|---|---:|---:|
| S | 5 | 15 000 ₽ |
| M | 10 | 30 000 ₽ |
| L | 20 | 60 000 ₽ |

**Условия (предложение — подстрой под себя):**
- Ставка ретейнера — та же, `3000 ₽/час` (в пакете).
- **Неизрасходованные часы** переносятся на следующий месяц (кап — 1 месяц),
  дальше сгорают.
- **Перерасход** сверх пакета — по той же ставке; **срочные/внеурочные** — ×1.5.
- Минимальный срок ретейнера — от 3 месяцев (окно стабилизации после внедрения).

**Сколько закладывать на сопровождение (эвристика).** На первые **3 месяца**
после внедрения бери пакет ≈ **10–15% часов внедрения в месяц** (стабилизация,
правки, вопросы). Дальше — по факту: активному клиенту обычно хватает S–M.

| Часы внедрения | Рекомендуемый пакет на старт |
|---|---|
| до 40 ч | S (5 ч/мес) |
| 40–100 ч | M (10 ч/мес) |
| 100+ ч | L (20 ч/мес) |

## 5. Что присылать в заявке (чтобы я посчитал)

Чем полнее — тем уже вилка. Минимум:

1. **Задача клиента** своими словами (что болит, что хочет на выходе).
2. **Банк(и)** и способ выписки: онлайн или файл; формат/образец (если файл).
3. **Что разносить:** сделки / счета / заказы; правило (по номеру, сумме, назначению).
4. **Смарт-процессы:** тариф Bitrix24 (есть ли), нужны ли кастомные поля/стадии.
5. **Уведомления:** куда (чат), что писать, фильтры.
6. **Развёртывание:** облако или **коробка/свой сервер**; чей сервер, доступы.
7. **Интеграции:** 1С / учёт / внешние системы — если да, какие.
8. **Срочность:** обычные сроки или горит.
9. **Объём истории:** нужна ли миграция/первичная загрузка.

Пришлёшь — верну: **часы (с разбивкой), фикс-цену с вилкой, срок, пакет сопровождения.**

## 6. Примеры расчёта

Ставки: аналитика 2100 / разработка 3000 / деплой 2600 ₽·ч. `×1.15` — только на
деплойную часть (контур клиента).

### Пример А: свой формат + правило разнесения + источник + деплой

**Заявка:** формат выписки клиента (файл, забираем из email по расписанию) + одно
кастомное правило разнесения + перенос на его сервер. Тариф со смарт-процессами есть.

| Задача | Часы | ×накл. | Ставка | Сумма |
|---|---:|---:|---:|---:|
| Анализ + ТЗ | 8.7 | — | 2 100 | 18 270 |
| Разбор формата | — | — | — | **0 (продукт)** |
| Источник файла (email по расписанию) | — | пакет | — | ~21 340 (≈ 800 Br) |
| Правило разнесения (**кастом** — тут ×1.2 уместен) | 13.0 | ×1.45 | 3 000 | 56 550 |
| Деплой (стандартный) | 6.5 | ×1.35 → ×1.15 | 2 600 | 26 234 |
| **Итого** | | | | **≈ 122 000 ₽** |

- **Стоимость:** фикс **~122 000 ₽ (≈ 4 570 Br)**, вилка **105–140 тыс.**
- **Срок:** ~2.5 недели.
- **Сопровождение:** пакет **M (10 ч/мес, 30 000 ₽)** на 3 мес.
- Разбор формата не в счёте — он бесплатный (раскатается на всех). Платное —
  индивидуальное: источник, кастом-правило, разворот.

### Пример Б: просто перенос Альфы на сервер (фикс-пакет)

Интеграция с Альфой **собрана** (транспорт/OAuth-ядро, реестр счетов, опрос) — для боевого пуска
нужны банк-креды клиента и прогон в его контуре. Это **фикс-пакет** (§2а) на перенос + настройку +
живой прогон, без почасовой сметы. (Статус компонентов — `docs/project-map.md`.)

| Задача | Часы | Ставка | Сумма |
|---|---:|---:|---:|
| Анализ/доступы | 3 | 2 100 | 6 300 |
| Деплой (compose готов): env, домен, TLS, токены, OAuth Альфы, вебхуки B24, смоук | 6.5 | 2 600 | 16 900 |
| Накладные ×1.35 + контур ×1.15 (на деплой) | — | — | +~10 000 |
| **Итого (~13 ч)** | | | **≈ 33 000 ₽ → пакет 35 000** |

- **Стоимость:** фикс **~35 000 ₽ (≈ 1 310 Br)**.
- **Срок:** ~3–5 рабочих дней.
- **Сопровождение:** пакет **S (5 ч/мес, 15 000 ₽)**.

> «Сырой»/нестандартный сервер → деплой по строке «сырой сервер» (17.3 ч) +
> смета сверху. Обучение/handover (+4 ч) — опционально.

### Пример В: новый формат файла

**Разбор самого формата — бесплатно** (продукт, раскатка на всех ~за месяц).
Платим только за то, что реально требует индивидуальной работы:

| Ситуация | Что платно | Цена |
|---|---|---:|
| Формату можно подождать общей раскатки, файл грузят вручную | ничего | **0** |
| Нужен свой **источник** файла (email/TG/чат/FTP по расписанию) | источник (1 тип) | **~800 Br (~21 000 ₽)** |
| Источник по **SFTP/SSH** (ключи) | источник с ключами | **~1 000 Br (~26 700 ₽)** |
| Нужен **раньше** общей раскатки / изолированный сервер | локальный разворот | редеплой **~290 Br** / первичный **~1 310 Br** |

- Типовой кейс «свой формат + забирать из email ежедневно, клиент уже на сервере»:
  `0 (формат) + 800 (источник) + 290 (редеплой) ≈ 1 090 Br (~29 000 ₽)`.
- Источник — **800 Br с каждого** клиента (первого, второго, любого): даром не
  отдаём, переиспользование транспорта = **наша маржа**.
- Стандартный 1С — уже поддержан → **0**.

### Пример Г: новый банк — понятный OAuth

| Компонент | BYN | RUB |
|---|---:|---:|
| Разработка (all-in, по нашим ядрам) | 1 600 | ~42 700 |
| Штатный разворот (первичный) | 1 310 | ~35 000 |
| **Итого** | **≈ 2 910 Br** | **≈ 77 700 ₽** |

Срок ~1.5–2 недели. Сопровождение S–M. *(Клиент уже на сервере → разворот = редеплой ~290 Br, итог ≈ 1 890 Br.)*

### Пример Д: новый банк — сложный (DCR/ключи, «Приор-класс»)

| Компонент | BYN | RUB |
|---|---:|---:|
| Разработка (сложный OAuth, стандарт) | 2 000 | ~53 400 |
| Менеджмент + ключи (DCR-регистрация, получение/разворот ключей, consent, sandbox) | 1 800 | ~48 000 |
| Небольшой риск (+10%) | 380 | ~10 100 |
| Штатный разворот (первичный) | 1 310 | ~35 000 |
| **Итого** | **≈ 5 500 Br** | **≈ 147 000 ₽** |

Срок **~5–6 недель** (растянуто ожиданием банка, не часами). Сопровождение M.
**Наценки ×1.2 нет** — банк стандартизован; сложность — в менеджменте/ключах/календаре.
