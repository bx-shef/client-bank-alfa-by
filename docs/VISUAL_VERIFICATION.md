# Визуальная верификация интерфейса

> Last reviewed: 2026-06-29

Правки UI / CSS / вёрстки не считаются выполненными, пока результат **не увидели
глазами**. LLM генерирует разметку вслепую — «собралось без ошибок» ≠ «выглядит
как надо». Полное обоснование и чек-лист — в базе знаний `ai-agent`,
`docs/16_visual-ui-verification.md`.

> Эта практика уже окупилась при инициализации репо: первый прогон скриншотов
> поймал тёмную тему, отрендеренную «чёрным по чёрному» (текст невидим) — баг,
> который не виден в коде и не ловится тестами/линтером.

## Как снять скриншоты

```bash
pnpm generate          # собрать статику в .output/public
pnpm screenshot        # снять все вьюпорты × темы → screenshots/
pnpm screenshot /      # только конкретный маршрут
```

`scripts/screenshot.mjs` поднимает локальный статик-сервер для `.output/public`
и снимает каждую страницу в двух вьюпортах (mobile 375px, desktop 1280px) и двух
темах (light/dark). Файлы пишутся в `screenshots/` (в `.gitignore`).

- **Браузер** — предустановленный Chromium окружения (`PLAYWRIGHT_BROWSERS_PATH`);
  `playwright install` запускать **не нужно**. Скрипт сам находит бинарь, поэтому
  переживает несовпадение версии npm-пакета `playwright` с предустановленной сборкой.
- **CI скриншоты не снимает** — `PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1` в `ci.yml`
  не даёт `playwright` тянуть браузер на установке.

## Минимум проверки

- Брейкпоинты: мобильный (375px) и десктоп — нет горизонтального скролла/переполнения.
- Темы: светлая и тёмная — текст читаем, контраст в порядке.
- Состояния: пусто / ошибка / длинный текст (когда появятся динамические данные).

> [!CAUTION]
> Скриншот реального портала клиента содержит данные CRM, домены, токены — его
> **нельзя** коммитить, класть в CI-артефакты или в облачный чат. Авто-скриншоты —
> только на dev/staging с мок-данными. Поэтому `screenshots/` в `.gitignore`.
