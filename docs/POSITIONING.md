# Позиционирование и отстройка от конкурентов

> Last reviewed: 2026-07-03

Бриф позиционирования приложения `shef.bankimport` в Маркете Bitrix24 с учётом
конкурентов. Опора для лендинга (#110), карточки Маркета
([`MARKETPLACE_LISTING.md`](MARKETPLACE_LISTING.md)) и данных для интеграторов
([`PARTNERS.md`](PARTNERS.md)).

---

## Конкурентное поле

### «Загрузка банка» (разработчик «Автоматизаторы») — прямой конкурент
Бесплатное приложение, зрелое. Уже умеет:
- привязку платежа к счёту, **перевод сделки в успех при полной оплате**;
- уведомления в чат со ссылками на компанию/счёт/сделку;
- поле «Платежи» в карточках компании/сделки/счёта;
- хранение всех платежей в смарт-процессе (тариф Professional+).

Ограничения (наши точки входа):
- только файл **1С `.txt`** в кодировке Windows-1251, других форматов нет;
- распознавание контрагента по **ИНН + КПП** (модель РФ);
- **только входящие** платежи;
- только облако, работа завязана на подписку.

### «Импорт банковской выписки» и прочие
Загрузка XLSX/CSV с маппингом колонок, деление приход/расход. Ручной, файловый,
без онлайн-подключения к банку и без РБ-специфики.

---

## Вывод: где мы сильнее, где нет

**Не воюем там, где конкурент силён.** Авторазнесение оплат (платёж закрывает
счёт/двигает сделку) конкурент отдаёт **бесплатно** — рынок воспринимает это как
базовую, а не премиальную функцию. Значит:

- базовое разнесение оплат должно быть в **бесплатном** приложении, иначе
  проигрываем в лобовом сравнении;
- платим не за сам факт разнесения, а за **то, чего у конкурента нет**.

**Настоящее УТП (бьём в пустое поле конкурента):**
1. **Белорусские банки и онлайн-выписка.** Альфа-Банк Беларусь и Приорбанк
   онлайн — у конкурентов только файл 1С. Портал может быть в любой стране.
2. **Распознавание по расчётному счёту, а не ИНН+КПП.** Работает для РБ, где КПП
   нет; для РФ-компаний с бизнесом в РБ — тоже.
3. **Больше форматов.** Онлайн + файл клиент-банк (`***** ^Type=`) + обмен 1С.
4. **Приходы и расходы**, а не только входящие.
5. **Свой сервер и код в git клиента — самостоятельный контур.** Приложение
   разворачивается целиком в инфраструктуре клиента: сервер клиента, исходный код
   в его git. Данные и токены доступа к банку **не уходят наружу**, нет зависимости
   от чужого облака и SaaS-подписки. Для финслужбы и служб безопасности это не
   «фича», а условие.

## Сегментация: РБ и РФ — разные сообщения

Один продукт, две аудитории с разной болью — говорить с ними надо по-разному.

### Основной сегмент — клиенты из РБ
Позиционируем в первую очередь на них. Сообщение — про рутину и локальные банки:
«Выписка Альфа-Банка Беларусь и Приорбанка попадает в CRM без ручного ввода,
контрагент находится по расчётному счёту». Это их ежедневная боль.

### РФ-компании с бизнесом в РБ — другой угол
Им не так важна рутина занесения — им важен **контроль над белорусским
подразделением**. Сообщение:
**«Проверяйте платежи ваших белорусских офисов прямо в своём Bitrix24»** —
головной офис в РФ видит движение по счетам белорусской «дочки» в единой CRM,
без пересылки выписок вручную. Здесь особенно играет п.5 (свой контур, данные под
контролем) — трансграничная передача финданных чувствительна.

## Формулировки позиционирования

**Одна фраза:** «Импорт выписки белорусских банков в Bitrix24 — онлайн из Альфы и
Приора или файлом, с распознаванием контрагента по расчётному счёту».

**Для РБ (основное):**
- «Выписка Альфа-Банка Беларусь и Приорбанка в CRM без ручного ввода».
- «Находим контрагента по расчётному счёту — работает для белорусских компаний».
- «Приходы и расходы, а не только поступления».

**Для РФ с бизнесом в РБ:**
- «Проверяйте платежи ваших белорусских офисов прямо в своём Bitrix24».

**Против конкурента (не называя его):**
- «Не только файл 1С — забираем выписку онлайн из белорусских банков».
- «Работаем в вашем контуре: сервер клиента, код в вашем git — данные и доступы
  к банку не уходят наружу, без SaaS-подписки».

## Влияние на дорожную карту

- **#109 (разнесение оплат):** пересмотреть границу free/платно. Базовое
  однозначное разнесение (1 кандидат, сумма сходится) — кандидат в бесплатную
  версию для паритета с конкурентом; платно — нестандартные правила, пограничные
  случаи, коробка, интеграции. Уточнить в issue.
- **Карточка Маркета:** усилить блок «белорусские банки + онлайн + по счёту» как
  главное отличие; не выпячивать разнесение как уникальное.
- **Лендинг (#110):** сохранить акцент на РБ-специфике и коробке; разнесение —
  как одна из возможностей, не как единственный крючок.
